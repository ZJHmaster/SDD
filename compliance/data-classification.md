# 数据分级分类标准 (Data Classification)

## 目的
建立统一的数据分级分类框架，使开发团队能够在编码、存储、传输和日志记录时做出正确的数据保护决策。

## 分级定义
详细定义参见 `standards/data-governance.md` 的第一章。以下为开发实践速查表：

| 级别 | 标记 | 代码中的识别方式 | 数据库列标记 |
|------|------|:----------------:|:-----------:|
| L4 绝密 | `@TopSecret` | 注解/装饰器标记 | `COMMENT '级别:L4'` |
| L3 机密 | `@Confidential` | 注解/装饰器标记 | `COMMENT '级别:L3'` |
| L2 内部 | `@Internal` | 注解/装饰器标记 | `COMMENT '级别:L2'` |
| L1 公开 | (默认) | 无需特殊标记 | 无 |

## 常见数据分级参考

### L4 绝密
- 系统主密钥 (Master Key)
- 数据加密密钥 (DEK/KEK)
- 核心算法/模型权重

### L3 机密
- 用户身份证号、护照号
- 银行卡号 (PAN)
- 用户登录密码（哈希后存储）
- 医疗健康记录
- 用户精确位置数据

### L2 内部
- 员工姓名、工号
- 内部系统配置参数
- 非公开的业务指标数据
- 用户邮箱和手机号

### L1 公开
- 产品名称和描述
- 公开的 API 文档
- 市场营销材料

## 开发层面的具体要求

### 日志中的脱敏规则
| 数据类型 | 脱敏方式 | 脱敏前 | 脱敏后 |
|---------|---------|--------|--------|
| 手机号 | 中间四位掩码 | 13812345678 | 138****5678 |
| 邮箱 | 用户名部分脱敏 | user@example.com | u***@example.com |
| 身份证 | 中间掩码 | 110101199001011234 | 110***********1234 |
| 银行卡 | 仅保留末四位 | 6225881234567890 | ************7890 |
| 密码 | 完全掩码 | myP@ssw0rd | ****** |

### API 返回中的脱敏
- 面向 C 端用户的接口返回 L3 数据时，必须进行脱敏处理。
- 面向内部运维后台的接口，需经权限校验后方可返回完整的 L3 数据，且须有操作审计日志。
