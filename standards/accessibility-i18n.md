# 国际化与无障碍规范 (Accessibility & i18n)

## 1. 国际化 (Internationalization - i18n)

### 1.1 文本外置化
- 所有面向用户的可见文本（UI标签、提示信息、错误消息）必须通过 i18n key-value 机制管理，禁止在代码中硬编写中文/英文字符串。
- 推荐 key 命名格式：`<module>.<page>.<element>.<action>`
  - 示例：`order.detail.button.cancel` → "取消订单" / "Cancel Order"

### 1.2 时区与日期
- 后端存储和传输一律使用 **UTC** 时间。
- 前端/客户端负责根据用户所在时区进行本地化展示。
- 日期格式严禁硬编码 `yyyy-MM-dd`，必须使用 `Intl.DateTimeFormat` 或等效本地化 API。

### 1.3 货币与数值
- 金额运算禁止使用浮点类型 (`float`/`double`)，必须使用整数分 (Cent) 或 `Decimal`/`BigDecimal`。
- 货币展示须标注货币代码（如 `CNY 199.00`, `USD 29.99`）。
- 数值千分位格式化须使用本地化格式器。

### 1.4 语言切换
- 应用须支持运行时语言切换，不需用户重新登录或刷新页面（SPA）。
- 语言包须支持按需加载 (Lazy Loading)，避免首屏加载全量翻译文件。

## 2. 无障碍 (Accessibility - a11y)

### 2.1 语义化 HTML
- 使用语义标签（`<nav>`, `<main>`, `<article>`, `<aside>`, `<header>`, `<footer>`），而非清一色 `<div>`。
- 表单元素必须关联 `<label>` 标签（通过 `for` 属性或包裹）。

### 2.2 ARIA 属性
- 交互组件（自定义按钮、模态框、下拉菜单）必须添加适当的 ARIA 角色和状态。
- 动态内容变更须通过 `aria-live` 属性通知屏幕阅读器。

### 2.3 键盘可达性
- 所有交互元素必须可通过键盘（Tab/Enter/Escape）访问和操作。
- 焦点指示器 (Focus Indicator) 不得被 CSS 强行移除 (`outline: none` 须替换为可视的替代样式)。

### 2.4 视觉对比度
- 文字与背景的对比度比值须满足 WCAG 2.1 AA 级标准（正文 ≥ 4.5:1，大号文字 ≥ 3:1）。
- 提供"高对比度"模式（可选）。

## 3. 合规驱动的适用范围
本规范对以下类型企业具有强制性约束力：
- 面向政府/公共服务的系统
- 需出海到欧美市场的产品（须满足 ADA Section 508 / EN 301 549）
- 面向金融/医疗等受监管行业的应用
