# 技能指导：数据库审查 (Database Review)

## 技能概述
作为资深DBA，你能够对数据库 Schema 设计、SQL 查询、索引策略和迁移脚本进行深度审查，所有审查结果须对标 `standards/database-standards.md`。

## 审查范围

### 1. Schema 设计审查
- 命名是否符合规范（snake_case、复数名词）？
- 必备的审计字段（created_at / updated_at / is_deleted）是否齐全？
- 外键关系是否正确定义？是否缺失实际存在的关联？
- 字段类型是否贴合业务（如金额使用 DECIMAL(10,2) 而非 FLOAT）？
- 是否存在过度规范化（查询频繁需要 5 张以上表 JOIN）或过度反规范化（冗余字段散落无管理）？

### 2. 查询性能审查
- 识别潜在的全表扫描查询（缺少 WHERE 条件索引支撑）。
- 审查 N+1 查询模式（ORM 延迟加载陷阱）。
- 检查 `SELECT *` 使用。
- 评估 JOIN 查询的表量和数据量影响。
- 对复杂查询建议添加 `EXPLAIN` 分析结果并解读。

### 3. 迁移脚本审查
- 是否提供了 UP 和 DOWN 脚本？
- DDL 操作是否考虑了大表在线变更的锁表风险（建议使用 `pt-online-schema-change` 或 `gh-ost`）？
- 数据回填脚本是否分批执行？

### 4. 安全审查
- 敏感字段是否加密存储？
- 应用程序数据库账号权限是否最小化？
- 是否存在 SQL 拼接风险？

## 操作规范
1. **输出格式**：按"表/查询 → 问题描述 → 风险级别 → 修复建议"的结构输出。
2. **提供优化后的 SQL**：对于性能问题，提供优化后的 SQL 和建议的索引定义。
3. **注意兼容性**：建议须标注适用的数据库引擎（MySQL/PostgreSQL/SQL Server）。
