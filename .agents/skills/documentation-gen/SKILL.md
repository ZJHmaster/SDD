# 技能指导：文档自动生成 (Documentation Generation)

## 技能概述
你能够阅读代码并自动生成多种类型的高质量技术文档，填补项目中长期缺失的文档缺口。生成的文档须直接符合 `standards/documentation-standards.md` 的规格要求。

## 生成能力

### 1. API 参考文档
- 阅读 Controller/Router 层代码，提取 HTTP 方法、路径、请求参数、响应结构。
- 输出格式为 OpenAPI 3.0 YAML 或 Markdown 格式的 API 参考页。
- 自动补充请求/响应示例。

### 2. 代码内注释 (JSDoc / JavaDoc)
- 分析函数签名、参数类型、返回值和异常抛出。
- 为缺少文档注释的公开方法/类生成标准化注释块。
- 注释内容侧重"函数做什么"和"参数/返回值的业务含义"，而非代码实现细节。

### 3. 架构概览文档
- 扫描项目目录结构和模块依赖关系。
- 生成包含 Mermaid 图表的架构概览文档。
- 涵盖模块职责描述、关键数据流和技术栈说明。

### 4. CHANGELOG 自动生成
- 基于 Conventional Commits 格式的 Git 提交历史，自动生成结构化变更日志。

### 5. README 自动更新
- 检查README中的环境变量列表、快速启动命令是否与实际代码一致。
- 自动生成或修正不一致的内容。

## 操作规范
1. **不臆造业务语义**：文档中的描述必须严格基于代码中可推断的信息。对于无法确定业务含义的部分，使用 `// TODO: 请补充业务描述` 标注。
2. **保留已有注释**：如果函数已有手写注释，保留并在其基础上增强，而非覆盖。
3. **支持增量更新**：如果文档已存在，仅更新与代码变更相关的部分。
