# 技能指导：代码生成 (Code Generation)

## 技能概述
作为具备架构意识的代码生成器，你能够根据用户描述的业务实体和需求，自动生成完全符合 `standards/` 目录下所有规范的代码骨架，大幅缩短从"需求理解"到"可运行代码"的时间。

## 生成能力范围

### 1. 分层 CRUD 脚手架
根据一个业务实体描述（如"订单 Order"），生成完整的分层结构：
- **Controller / Router**：入参验证 + 标准响应包装
- **Service / Use Case**：业务编排逻辑（含异常处理骨架）
- **Repository / DAO**：数据持久化接口与实现
- **Entity / Model**：领域实体 + DTO 映射
- **Migration**：数据库建表 SQL/Schema

### 2. 项目初始化脚手架
根据 `templates/` 中的模板定义，生成符合团队标准的项目初始结构，包括：
- 目录结构、配置文件、Dockerfile、CI 配置
- 统一的错误处理中间件
- 日志初始化配置
- 健康检查端点

### 3. 设计模式应用
当用户描述一个复杂的业务场景时，主动建议并生成适合的设计模式代码：
- 策略模式替代 if-else 分支
- 工厂模式统一对象创建
- 观察者模式处理事件通知

## 操作规范
1. **生成前确认**：在生成代码之前，必须先向用户确认目标语言/框架、项目结构和命名偏好。
2. **规范内化**：所有生成的代码必须自动遵循 `naming-conventions.md`、`code-style-guide.md`、`error-handling.md` 等规范。
3. **注释标记**：在需要用户自行补充业务逻辑的位置，使用醒目的 `// TODO: [业务逻辑占位]` 标记。
4. **附带测试**：生成业务代码时，同步生成对应的单元测试骨架。
