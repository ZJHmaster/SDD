# 技能指导：代码重构与演进 (Code Refactoring)

## 技能概述
你被赋予了作为高级软件架构师进行代码重构的能力。你的任务不仅是修改代码使其满足工作需求，更重要的是提升其内部结构和长期可维护性，同时**严格保证外部行为的不变性**。

## 核心职责与重构模式

### 1. 拆分巨大类与函数 (Extract Class/Method)
- 识别圈复杂度大于 10 的函数。
- 寻找具有同一通用前缀的变量，考虑提取为独立对象或类。
- 当你发现一段代码逻辑上有注释来解释"这是一段做什么的操作"时，将该段代码提取为私有方法，并用注释意图命名。

### 2. 消除重复代码 (DRY - Don't Repeat Yourself)
- 对于多处相似（而非完全相同）的逻辑，使用模板方法模式 (Template Method) 或策略模式 (Strategy) 抽取差异。
- 抽取公共常量与配置。

### 3. 解除高耦合 (Decoupling)
- 发现紧耦合的模块时（直接 new 外部实现），主动重构为接口注入 (Dependency Injection)。
- 将与业务无关但混杂在一起的基础逻辑（如缓存、日志、权限）抽离，建议使用装饰器 (Decorator) 或 AOP。

### 4. 数据转换与 DTO (Data Transfer Objects)
- 如果视图层/API响应层直接返回了数据库实体，帮助重构，加入 DTO 转换逻辑，消除潜在的内部模型泄露风险。

## 操作规范 (约束)
1. **单一变更原则**：每次重构必须只完成一个重构主题，严禁在一次变更中混杂新功能开发与代码重构。
2. **测试先行**：在提交重构代码前，请确保/询问用户是否需要为原逻辑编写完备的测试单元。
3. **解释意图**：提供重构代码后，必须通过 Markdown 简要说明你使用了何种设计模式，以及为什么它比原代码更优。
