# 需求追溯工作流 (Requirement Traceability)

自动扫描代码库中的 `@bizReq` 追溯标记，与业务文档进行交叉比对，维护追溯矩阵并产出覆盖率报告。

## 步骤指南

1. **业务文档清单提取**
   - 扫描 `docs/biz/` 目录下的所有 BRS 文档。
   - 解析 YAML front matter 中的 `docId`、`traceability` 字段。
   - 构建"需求 → 预期关联资产"的清单。

2. **代码层标记扫描**
   - 在源代码目录中使用 `grep_search` 扫描 `@bizReq`、`BizReq:` 等标记。
   - 提取每个代码文件关联的需求 ID。
   - 构建"代码模块 → 实际关联需求"的反向索引。

3. **测试层标记扫描**
   - 在测试目录中扫描 `@bizReq`、`@acceptCriteria` 标记。
   - 提取每个测试用例关联的需求 ID 和验收标准编号。

4. **交叉比对与缺口识别**
   - 比对"需求清单"与"代码关联"和"测试关联"：
     - **需求无代码**：需求文档存在，但没有任何代码模块关联了该需求 → 可能未实现或漏标。
     - **需求无测试**：需求有代码实现但缺乏测试覆盖 → 测试缺口。
     - **代码无需求（孤立代码）**：代码模块没有关联任何需求 → 可能是技术债或遗留代码。
     - **文档过期**：需求文档标记的 `traceability.modules` 在代码中已不存在 → 文档未同步更新。

5. **追溯覆盖率报告产出**
   ```markdown
   # 需求追溯覆盖率报告 - 2026-04-26

   ## 总体指标
   - 需求总数: 42
   - 需求覆盖率 (有代码关联): 38/42 = 90.5%
   - 测试追溯率 (有测试关联): 35/42 = 83.3%
   - 孤立代码模块数: 5

   ## 缺口清单
   | 需求ID | 问题 | 建议 |
   |--------|------|------|
   | BRS-PAY-2026-003 | 无代码关联 | 确认是否已实现 |
   | BRS-ORD-2026-007 | 有代码无测试 | 补充测试用例 |

   ## 孤立代码清单
   | 代码模块 | 文件 | 建议 |
   |---------|------|------|
   | LegacyExporter | src/utils/legacy-export.ts | 确认是否仍需要 |
   ```

6. **追溯矩阵自动更新**
   - 基于扫描结果，自动更新 `docs/traceability/` 下的矩阵文件。
   - 向用户确认后执行更新。
