# 性能剖析工作流 (Performance Profiling)

当系统出现性能退化或需要进行性能优化时触发。

## 步骤指南

1. **问题收集与基线建立**
   - 明确性能问题的现象描述（哪个接口慢？慢多少？P50 还是 P99？）。
   - 确认当前的性能基线数据（参照 `standards/performance-standards.md`）。
   - 确定分析范围（单一接口 / 整个服务 / 全链路）。

2. **代码级热点分析**
   - 调用 `code-optimization` 技能，审查目标代码的算法复杂度。
   - 识别 N+1 查询、不必要的序列化/反序列化、阻塞 I/O 等问题。
   - 利用 `performance-analysis` 技能进行系统级瓶颈定位。

3. **数据库级分析**
   - 调用 `database-review` 技能，审查相关查询的执行计划。
   - 评估索引有效性和查询优化空间。
   - 检查连接池配置是否合理。

4. **优化方案制定**
   - 按"投入产出比"排序优化措施：
     - 🟢 Quick Win：修改配置/加索引/调参数 → 预计 1 天内完成
     - 🟡 中等投入：代码重构/缓存引入 → 预计 3-5 天
     - 🔴 大型优化：架构调整/分库分表/引入新组件 → 需按项目规划

5. **验证与回归**
   - 优化完成后进行性能回归测试。
   - 对比优化前后的指标数据。
   - 更新性能基线文档。
