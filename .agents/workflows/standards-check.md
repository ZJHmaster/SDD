# 规范化与标准检查工作流 (Standards Check)

此工作流用于日常的高频自动审查，它是所有 AI Agent 进行后续评审和合并分析的第一道防线，确保低级错误被挡在门外。

## 触发点与步骤

1. **获取最新规范字典**
   - 代理挂载 `standards/` 目录下的所有文件并加载至认知上下文（包含风格、命名、安全等规则）。

2. **多语言智能匹配**
   - 识别当前检查的代码文件类型或代码片段语言，根据对应的特性匹配检查清单。
     - 若为前端文件：着重依据规范检查组件解耦度、Hooks使用合规性、CSS层叠层级等。
     - 若为服务端文件：执行 `api-design-guide` 和 分层架构规则的比对。

3. **坏气味快速拦截标记**
   - 不生成冗长的宏观叙事报告代码，而是直接**通过行内注释精准指向**。
   - 格式输出样例："Line 42：违背 `naming-conventions` 第一条，建议提取为具有表达性的名称，而非简单的 `let res = ...`"。

4. **强制性修复脚本下发**
   - 如果能够直接确定确切的规整化修改（例如错误的换行/花括号未对齐/命名格式不对），则通过提供的底层终端修正补丁或者询问用户是否一键覆写源文件修补完毕。
