# 项目健康度检查工作流 (Project Health)

定期（建议每周/每迭代）运行，产出项目技术健康度的全景量化报告。

## 步骤指南

1. **代码质量维度**
   - 扫描全项目，统计以下指标：
     - 文件数量与代码行数 (LOC)
     - 平均圈复杂度 (Cyclomatic Complexity)
     - 超过 300 行的"大文件"清单
     - 超过 50 行的"大方法"清单
     - `TODO/FIXME` 总数及分布热力图
     - 重复代码率估算

2. **测试健康度维度**
   - 统计测试文件数量与预估覆盖率。
   - 识别核心业务模块（Domain 层）的测试缺失区域。
   - 标记最近 30 天新增的未覆盖代码。

3. **依赖健康度维度**  
   - 检查过期依赖数量。
   - 标记已知安全漏洞。
   - 评估依赖树深度和冗余依赖。

4. **文档健全度维度**
   - README 完整性评分。
   - 公开 API 文档覆盖率。
   - ADR 记录是否持续维护。
   - CHANGELOG 更新是否及时。

5. **架构健康度维度**
   - 分层违规检测（是否有 Controller 直接引用 Repository？）。
   - 循环依赖检测。
   - 模块粒度评估（过大/过小的模块）。

6. **综合报告产出**
   - 以 Markdown Artifact 格式产出《项目健康度报告 - YYYY-MM-DD》。
   - 使用数值评分 (1-10) 对各维度打分，形成总体健康指数。
   - 与上期报告对比，标示趋势变化（↑ 改善 / ↓ 恶化 / → 持平）。
   - 列出 Top 5 最急迫的改善建议。
