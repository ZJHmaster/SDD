# 自动化代码评审工作流 (Code Review)

该工作流用于在代码合并前/提交后，利用 AI 代理能力进行自动化代码审查。

## 步骤指南

1. **环境与变更分析**
   - 代理使用代码差异识别工具或读取 Git diff 了解本次待审查的改动集。
   - 提取代码修改意图，确认修改的范畴是否匹配描述的需求。

2. **核心指标审查**
   - **规范性检查**：调用 `standards/code-style-guide.md` 和 `standards/naming-conventions.md` 等文档，扫描改动是否符合团队标准。
   - **安全性检查**：应用 `error-detection` Skill 检查改动是否引入了注入点、越权等风险。
   - **架构一致性**：确认代码变更是正确存放在相应的架构分层（如：业务逻辑是否错误写在了接口层）。

3. **生成评审报告**
   通过 Artifact 生成规范的评审报告文件：
   - 包含【变更总结】、【亮点赞赏】、【改进建议】三部分。
   - 对发现的缺陷按严重级排序。
   - 对于非致命缺陷，附带优化代码参考 `diff` 片段。

4. **代码调整互动**
   如果使用者同意评审意见，AI 可以直接帮助执行重构修改操作并应用至本地代码。
