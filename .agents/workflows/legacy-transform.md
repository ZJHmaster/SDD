# 存量代码改造自动化工作流 (Legacy Transform)

针对历史遗留项目进行安全、渐进式现代化改造的工作流程。要求在保证业务不中断的前提下提升代码质量。

## 步骤指南

1. **改造前环境冻结与保护**
   - 确定并锁定当前重构的范围。
   - 优先通过生成 AI Skill 进行测试覆盖：如果模块缺乏测试，利用 AI 基于旧有业务逻辑提取和生成测试用例并运行通过，形成行为保护网。

2. **缺陷及异味清除（微观级别）**
   - 自动应用 `code-refactoring` 及其消除坏味道的能力，先从微小的函数拆分、内联临时变量开始。
   - 这阶段着重将面条代码拆解为职责单一的小模块，解耦对静态变量和全局状态的变异操作。

3. **模型和架构提权 (宏观级别)**
   - 发现并隔离出"防腐层"（Anti-Corruption Layer）。在改造不同系统交互的地方，抽取统一的接口桥接老系统。
   - 将强行聚合在一块的系统，基于领域概念边界分离成新服务或模块独立。

4. **改造确认与持续集成**
   - 将新代码逻辑和旧逻辑使用特性开关（Feature Toggle）管理，建议提供双写配置（影子流量回放）等重度灰度策略建议方案。
   - 产出改造前后的结构以及数据流向对比 Diff 与状态报告。
