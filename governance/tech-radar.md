# 技术雷达 (Tech Radar)

## 概述
技术雷达是团队/公司对技术选型的共识管理工具。它将技术划分为四个象限和四个环，帮助团队在技术选型时做出一致的决策，避免技术栈碎片化。

## 象限 (Quadrants)
| 象限 | 含义 |
|------|------|
| **语言与框架** | 编程语言、应用框架、UI 框架 |
| **工具** | 开发工具、构建工具、测试工具、IDE 插件 |
| **平台** | 云服务、数据库、消息队列、容器平台 |
| **技术与方法** | 架构模式、设计方法、工程实践 |

## 环 (Rings)
| 环 | 含义 | 行动 |
|----|------|------|
| 🟢 **Adopt (采纳)** | 成熟可靠，团队有丰富经验 | 新项目默认使用 |
| 🔵 **Trial (试用)** | 有明确价值，值得在受控项目中尝试 | 可在非核心项目使用 |
| 🟡 **Assess (评估)** | 有潜力但尚需评估 | 进行 PoC 或技术调研 |
| 🔴 **Hold (暂缓)** | 不推荐使用（过时/有更好替代/风险高） | 不用于新项目；存量项目计划迁移 |

## 雷达示例 (请根据企业实际情况调整)

### 语言与框架
| 技术 | 环 | 备注 |
|------|--:|------|
| TypeScript | 🟢 Adopt | 前后端统一类型安全 |
| Java 17+ / Spring Boot 3 | 🟢 Adopt | 企业主力后端 |
| React 18+ | 🟢 Adopt | 前端主力框架 |
| Next.js / Nuxt.js | 🔵 Trial | 全栈框架 SSR |
| Rust | 🟡 Assess | 高性能基础设施组件 |
| jQuery | 🔴 Hold | 新项目禁止使用 |

### 工具
| 技术 | 环 | 备注 |
|------|--:|------|
| ESLint + Prettier | 🟢 Adopt | 前端代码质量 |
| GitHub Actions / GitLab CI | 🟢 Adopt | CI/CD 管线 |
| k6 | 🔵 Trial | 性能测试 |
| AI Coding Assistant | 🔵 Trial | 配合 SDD 规范使用 |

### 平台
| 技术 | 环 | 备注 |
|------|--:|------|
| PostgreSQL | 🟢 Adopt | 主力关系型数据库 |
| Redis | 🟢 Adopt | 缓存与会话管理 |
| Kubernetes | 🟢 Adopt | 容器编排 |
| Kafka | 🔵 Trial | 高吞吐事件流 |

## 更新节奏
- 技术雷达每季度由架构委员会 (Architecture Board) 评审更新一次。
- 任何团队成员可提出"技术雷达变更提案"（RFC 形式），经讨论后纳入。
