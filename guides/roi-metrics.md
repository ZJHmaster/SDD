# 投资回报率度量框架 (ROI Metrics)

## 概述
为了证明 SDD 框架落地的价值，需要建立可量化的度量体系。本文档提供推荐的指标体系和计算方法。

## 1. DORA 四项关键指标
Google DevOps Research and Assessment 提出的行业标准工程效能指标：

| 指标 | 定义 | Elite 标准 | 度量方式 |
|------|------|----------|---------|
| **部署频率** | 多久向生产环境部署一次 | 按需（每天多次） | CI/CD 平台数据 |
| **变更前置时间** | 从代码提交到成功部署的耗时 | < 1 小时 | 代码库 + CI/CD 数据 |
| **变更失败率** | 部署导致故障的比例 | 0-15% | 故障管理系统数据 |
| **故障恢复时间 (MTTR)** | 从故障发生到恢复的耗时 | < 1 小时 | 监控系统数据 |

## 2. 质量维度指标

### 缺陷密度
```
缺陷密度 = 线上缺陷数 / 代码行数 (KLOC)
目标: < 1 个缺陷 / KLOC
```

### 测试覆盖率
```
核心域覆盖率 ≥ 80%
整体覆盖率 ≥ 60%
```

### 代码评审效率
| 指标 | 落地前基线 | 落地后目标 |
|------|----------|----------|
| 评审周转时间 | 24-48 小时 | < 4 小时 |
| 每次评审发现的问题数 | 1-2 个 | 5-10 个 (AI+人工) |
| 低级错误漏检率 | 30%+ | < 5% |

## 3. 效率维度指标

### 新项目启动效率
| 指标 | 落地前 | 落地后 |
|------|-------|-------|
| 从零到可运行骨架的时间 | 2-3 天 | < 30 分钟（使用模板） |
| 基础配置一致性 | 每人不同 | 100% 一致 |

### 新人上手时间
| 指标 | 落地前 | 落地后 |
|------|-------|-------|
| 新人首次有效贡献时间 | 2-4 周 | 3-5 天 |
| 新人需要的一对一指导时间 | 40+ 小时 | < 10 小时 |

## 4. 安全维度指标
| 指标 | 落地前 | 落地后 |
|------|-------|-------|
| 安全漏洞平均发现时间 | 上线后数周 | 代码提交时 |
| 高危漏洞修复周期 | 1-2 周 | < 24 小时 |
| 合规检查通过率 | 手动/无法度量 | 自动化 + 可追溯 |

## 5. ROI 计算公式

### 节省的成本
```
年度节省 = 
  (减少的线上缺陷数 × 每个缺陷的平均修复成本)
+ (节省的评审时间 × 开发者人力成本)
+ (避免的安全事故 × 事故平均损失)
+ (新人上手时间缩短 × 年入职人数 × 人力成本)
```

### 投入的成本
```
投入成本 = 
  框架初始建设时间 × 人力成本
+ 培训时间 × 参与人数 × 人力成本
+ 日常维护时间 × 人力成本
```

### ROI
```
ROI = (年度节省 - 投入成本) / 投入成本 × 100%
```

## 6. 度量节奏
| 频率 | 度量内容 |
|------|---------|
| 每次发版 | 变更失败率、部署频率 |
| 每周 | 评审周转时间、CI 管线通过率 |
| 每月 | 缺陷密度、测试覆盖率、依赖漏洞数 |
| 每季度 | DORA 综合评估、成熟度模型自评、ROI 计算 |
