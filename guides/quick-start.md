# 快速上手指南 (Quick Start)

欢迎使用企业级 AI 编程范式解决方案 (Software Development Discipline, SDD)。本指南将帮助你在本地快速接入和使用这套规范体系，充分发挥 AI 智能代理的作用。

## 1. 概览
SDD 并不是束之高阁的文档，它已经被编写成了结构化约束，深度融于开发环境，能够被 AI Agent 识别与执行。
整个系统分为三大块：
- `/standards`：静态编码与架构规定
- `/.agents/skills`：赋予 AI 开发助手的专业检视与整改能力
- `/.agents/workflows`：一键触发的自动化多合一工程流程

## 2. 如何启动智能代码评审
利用系统内置的 agent 工作流程提升代码质量。

1. **在你修改完代码需要自测或提交 Pull Request 前**。
2. 呼叫你的 AI 开发助手（例如在对话框中输入）。
3. 键入斜杠命令触发评审流程：`/code-review`。
4. AI 会自动寻找你最近变更的文件，根据 `/standards` 里的安全、命名和架构规范给你出具有针对性的修正建议乃至帮你一键重构。

## 3. 进行遗留项目改造探索
当你刚接手一个存在技术债的庞大方法或老旧模块时：
1. 告诉 AI Agent：“请基于业务安全基线，使用遗留项目改造工作流对这一模块进行重组。”
2. 内部会激活 `/legacy-transform` 流程和相关的重构技巧，为您逐步补齐测试并安全解耦。

## 4. 个性化环境调整
- 你可以基于当前的 `.agents/skills` 定制适合你业务中特定框架的特异化 AI 技能。
- 修改 `/standards` 下的文件，AI 会自动吸收并在接下来的检查中实施你的新标准。
